ROOT CAUSE ANALYSIS
===================

The issue is in the FFmpeg command configuration:

Current (BROKEN):
  -movflags frag_keyframe+empty_moov
  ↓
  Produces MP4 with empty moov atom (missing codec parameters)
  ↓
  MediaSource API cannot parse initialization segment
  ↓
  sourceopen event never fires
  ↓
  SourceBuffer cannot be created/used

SOLUTION
========

Change FFmpeg command line in /home/user/shellyclient/index.js

Line 207, change from:
  '-movflags', 'frag_keyframe+empty_moov',

To:
  '-movflags', 'frag_keyframe+delay_moov',

Why this works:
  • delay_moov: Includes full moov atom with first segment
  • Browser receives valid H.264 initialization packet
  • MediaSource API can parse codec parameters
  • sourceopen event fires correctly
  • SourceBuffer can append data
  • Video displays in player

Impact:
  • No server changes needed
  • No browser client changes needed
  • Simple 1-line fix
  • Full H.264 video streaming will be operational
